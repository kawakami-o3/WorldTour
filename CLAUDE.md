# WorldTour プロジェクト

世界の国々の政情・地理・歴史などを紹介するナレッジベース。

## プロジェクト構成

```
WorldTour/
├── CLAUDE.md          # エージェント向けルール・設定
├── articles/          # 日別の記事ファイル
│   └── YYYY-MM-DD.md  # 日付ごとの記事
└── INDEX.md           # 目次ページ（全記事の一覧）
```

## ルール

### 記事の作成
- 調査・紹介した内容は `articles/YYYY-MM-DD.md` に書き出す
- 1日に複数トピックを扱った場合は同じ日付ファイルにまとめる
- 既に同日のファイルがある場合は追記する

### 記事のフォーマット
- 見出しに国名・地域名を含める
- 基本情報（首都、人口、面積、公用語、通貨など）を冒頭に記載する
- 地理・歴史・政治・文化など多角的に紹介する
- 情報の正確性に注意し、不確かな点は明記する
- `**` による太字強調は使用しない。重要性は文章表現（語順、修飾語、文脈の補足など）で伝える

### 目次の更新
- 記事を追加・更新したら `INDEX.md` を必ず更新する
- 目次には日付、取り扱った国・地域名、トピックの概要を記載する
- 目次は日付の降順で並べる

### 調査・情報収集
- まず検索段階を英語に限定しないこと。
- 検索クエリは、英語、現地公用語、必要なら周辺国の主要言語でも作成すること。
- 国名だけでなく、州・県・地方都市・国境地帯・主要組織名・少数民族名・旧称・別表記も用いること。
- 英語の国際媒体だけで構成しないこと。
- 最低でも、現地語の地方媒体3件、現地語の全国媒体2件、隣国または利害関係国の報道1件、国際報道2件以下、公的資料1件を含めること。
- 各ソースについて、言語、国、媒体種別、原題、見出しの直訳、要約、ローカル性の根拠を示すこと。
- 原文に重要な政治語彙・歴史語彙がある場合は、その原語と訳語候補を示すこと。
- 最終出力は以下の順で構成すること:
  1. 今日のローカルな異変
  2. なぜ地元では大きいのか
  3. 歴史・地理・国際関係の背景
  4. 英語報道では見えにくい点
  5. 不確実な点とソース間の不一致
- 最後に監査表として、使用ソース総数、言語別内訳、地方媒体比率、英語媒体比率、未検証論点を出すこと。

### ツール
- WebSearchツールはこの環境では使用できない（API制限）
- Web上の情報取得にはAgent経由のWebFetchを第一手段として使う
- WebFetchが403/429エラー等で失敗した場合は、agent-browserで再試行する

### 言語
- 記事・目次はすべて日本語で記述する
